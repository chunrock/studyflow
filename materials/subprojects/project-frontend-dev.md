# project-frontend-dev 하위 프로젝트

## 목적

Figma 또는 디자인 명세를 React 기반 프론트엔드로 변환하기 위한 전용 템플릿이다.

## 적용 대상

- 웹앱 화면 구현
- 디자인 시스템 반영
- 정적 프로토타입에서 실제 React 컴포넌트로 전환
- UI 문서화와 코드 리뷰가 필요한 프론트엔드 프로젝트

## 기본 역할

| 에이전트 | 경로 | 담당 |
| --- | --- | --- |
| `team-lead@project-frontend-dev` | `<PROJECT_ROOT>/frontend` | 작업 분해, 구현 조율, 리뷰 요청 |

## 함께 쓰는 팀

- `design-team`: Figma 분석, UI/UX, 시각 기준 정리
- `dev-team`: React 구현, 상태관리, 컴포넌트 구조 설계
- `qa-team`: 접근성, 반응형, 콘솔 에러, 회귀 테스트 확인

## 적용 전 채울 값

- `<PROJECT_ROOT>/frontend`
- 디자인 원본 링크 또는 파일 경로
- 사용하는 프레임워크
- 디자인 토큰 위치
- 고정 개발 서버 포트
- 스크린샷 검증 기준

## 권장 작업 흐름

1. 디자인 기준과 기존 컴포넌트 구조를 확인한다.
2. 화면 단위를 컴포넌트와 데이터 단위로 나눈다.
3. 구현 후 브라우저 스냅샷과 콘솔 에러를 확인한다.
4. 모바일/데스크톱 주요 뷰포트를 검증한다.
5. 변경 이력과 남은 이슈를 기록한다.
