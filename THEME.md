# THEME.md — 테마 디자인 기본 규칙

이 문서는 웹 자료를 생성할 때 적용하는 **공통 테마 기준**입니다. 기준 원본은 **검진팀 대시보드**(`검진팀 대시보드/dashboard/web/styles/`)이며, 이 테마는 건강검진 제안서 자동생성기의 테마를 계승한 것입니다. 새 프로젝트·새 화면은 아래 규칙을 기본값으로 따릅니다.

---

## 1. 파일 구조: 토큰과 컴포넌트를 분리한다

```
web/
├─ styles/
│  ├─ theme.css   ← 디자인 토큰(색·폰트·레이아웃 변수)만. 이 파일만 고치면 전체 테마가 바뀐다.
│  └─ app.css     ← 컴포넌트 스타일(카드·버튼·표·차트 등)
├─ fonts/         ← 번들 폰트(TTF). 외부 CDN 폰트를 쓰지 않는다.
└─ index.html
```

- 색상·치수는 **반드시 `:root` CSS 변수로 정의**하고, 컴포넌트 CSS에서는 변수만 참조한다. 하드코딩 색상 금지.
- 인라인 스타일로 테마 값을 넣지 않는다.
- 프로젝트에 기존 `theme.css`가 있으면 먼저 그 파일의 토큰을 따른다. 표준 토큰으로 교체하는 작업은 별도 요청이 있을 때만 한다.

## 2. 디자인 토큰 (표준값 — 검진 대시보드 기준)

```css
:root{
  /* 색상 */
  --blue:#2878d0;   /* 주 색상(브랜드 블루) — 주요 버튼·강조 */
  --deep:#15549b;   /* 진한 블루 — 제목 숫자·그라디언트 끝 */
  --cyan:#5cc8e8;   /* 보조 시안 — 차트 보조 계열 */
  --accent:#a5e4f4; /* 밝은 액센트 — 어두운 배경 위 강조 텍스트 */
  --paper:#f7fbfe;  /* 카드 내부 옅은 면 */
  --pale:#eaf3fc;   /* 옅은 블루 면 — 리스트 행·호버 배경 */
  --ink:#172534;    /* 본문 텍스트 */
  --muted:#6b7885;  /* 보조 텍스트 */
  --line:#d7e1eb;   /* 테두리·구분선 */
  --bg:#eef5fa;     /* 페이지 배경 */

  /* 레이아웃 */
  --sbw:90px;       /* 사이드바 폭 */
  --tbh:70px;       /* 상단바 높이 */
}
```

- 그라디언트: 상단바·강조 카드에 `linear-gradient(135deg,#0e3e78,#2878d0)`, 사이드바에 `linear-gradient(180deg,#2878d0,#15549b)`.
- 상태 색이 필요하면 프로젝트 토큰으로 추가하되(예: `--red`), 위 팔레트와 어울리는 톤을 쓴다.

## 3. 타이포그래피

- 폰트: 번들한 **Noto Sans KR** (`fonts/NotoSansKR-Regular.ttf` 400, `NotoSansKR-Bold.ttf` 700–900)을 `@font-face`로 선언하고 `font-display:block`을 쓴다. 오프라인·인쇄 환경에서도 동일하게 보이게 하기 위함이다.
- 폰트 스택: `"Proposal Noto Sans KR","Noto Sans KR","Apple SD Gothic Neo",sans-serif`
- 기본: `line-height:1.4`, `-webkit-font-smoothing:antialiased`
- 위계(대시보드 기준): 페이지 제목 21px/900, 카드 제목 14px/800, KPI 숫자 32px/900(`--deep`, `letter-spacing:-.02em`), 본문 13px, 보조·라벨 12px(`--muted`, 600–700).
- 굵기를 폭넓게 쓴다: 라벨류는 600–800, 숫자·제목은 800–900.

## 4. 레이아웃 골격

- **상단바(topbar)**: 고정, 높이 `--tbh`, 블루 그라디언트, 흰 제목(21px/900) + 우측 유틸(빵부스러기·인쇄 버튼).
- **사이드바(sidebar)**: 고정, 폭 `--sbw`, 세로 그라디언트, 아이콘 전용 내비게이션(54px 정사각, radius 14px, active 시 `#ffffff33` 배경 + 좌측 흰 인디케이터 바, 호버 툴팁).
- **콘텐츠(content)**: 페이지 배경 `--bg`, 카드 그리드로 구성.
- 반응형: 좁은 화면에서 패딩·제목 크기만 축소(모바일 전용 레이아웃은 요청 시에만).

## 5. 컴포넌트 규칙

- **카드**: 흰 배경, `border-radius:16px`, 그림자 `0 8px 22px #15549b14`, 패딩 `20px 22px`.
- **KPI 카드**: 라벨 13px/`--muted` → 값 32px/900/`--deep` → 부가설명 12px. 강조 KPI는 블루 그라디언트 배경 + 흰 글자 + `--accent` 라벨.
- **버튼**: `border-radius:10px`, 패딩 `12px 20px`, 굵기 800.
  - `primary`: `--blue` 배경 + 흰 글자
  - `ghost`: 흰 배경 + `--line` 테두리 + `--deep` 글자
  - 상단바 안 버튼은 알약형(`border-radius:999px`) + 반투명 흰색(`#ffffff22`/`#ffffff77` 테두리).
- **리스트 행·칩**: `--pale` 배경, radius 9px 내외, 이름은 800/`--ink`, 메타는 600/`--muted`.
- **표·차트**: 카드 안에 배치. 차트 SVG는 가로 스크롤 허용(`overflow-x:auto`).

## 6. 아이콘·곡률·배치

색상 팔레트는 유지하고, 아이콘과 컨트롤의 형태만 최신 Apple 계열 인터페이스처럼 부드럽고 정돈된 인상으로 맞춘다. 특정 Apple 자산을 복제하지 말고, 아래 원칙을 범용 UI 규칙으로 적용한다.

- **아이콘 스타일**: 선형 아이콘을 기본으로 하고, stroke는 2px 안팎의 균일한 두께를 쓴다. 아이콘끼리 선 두께, 끝 모양, 채움 여부를 섞지 않는다.
- **아이콘 크기**: 일반 버튼·메뉴 아이콘은 20–22px, 사이드바 주요 아이콘은 24–26px를 기본값으로 한다. 같은 영역 안에서는 아이콘 박스 크기를 고정해 텍스트 길이나 hover 상태가 레이아웃을 흔들지 않게 한다.
- **광학 정렬**: 아이콘은 수학적 중앙보다 시각적 중앙을 우선한다. 화살표·재생·인쇄처럼 한쪽으로 무게가 쏠리는 형태는 1–2px 보정해 버튼 안에서 안정적으로 보이게 한다.
- **곡률**: 아이콘 버튼과 작은 컨트롤은 정사각형 모서리를 피하고 연속적인 둥근 사각형 느낌을 준다. 40–44px 컨트롤은 `border-radius:12px`, 48–56px 주요 아이콘 버튼은 `border-radius:14px–16px`를 기준으로 한다.
- **배치**: 아이콘과 텍스트가 함께 있는 버튼은 아이콘 20px, 간격 8px, 텍스트 순서로 배치한다. 아이콘 전용 버튼은 같은 행에서 동일한 정렬선과 동일한 터치 영역을 유지한다.
- **상태 표현**: hover·active·selected는 색상 변경만으로 처리하지 말고 배경 면, 투명도, 미세한 그림자, 1px 이동 중 하나를 함께 사용한다. 과한 확대 애니메이션은 쓰지 않는다.
- **계층감**: 강조 아이콘은 기존 `--blue`, `--deep`, `--accent` 범위 안에서 단색 또는 계층형 투명도만 사용한다. 새로운 색상 계열을 추가하지 않는다.
- **모션**: 상태 전환 아이콘은 120–180ms의 짧은 easing으로 회전·페이드·위치 이동만 사용한다. 의미 없는 반복 애니메이션은 피하고, 저장 중·동기화 중·완료처럼 상태를 설명할 때만 쓴다.
- **배경 효과**: 유리감이나 반투명 효과가 필요하면 기존 색상 위에 `backdrop-filter`, 흰색/블루 계열의 낮은 투명도, 얇은 테두리만 사용한다. 색상 토큰 자체를 바꾸지 않는다.
- **아이콘 선택**: 기능이 명확한 곳은 익숙한 아이콘을 우선한다. 추상적이거나 혼동될 수 있는 기능은 아이콘만 두지 말고 텍스트 라벨을 함께 제공한다.

참고 방향: Apple SF Symbols는 텍스트와 자연스럽게 정렬되는 심볼, 여러 weight/scale, 계층형 렌더링, 상태 변화 모션을 제공한다. 이 테마에서는 그 방향성을 웹 표준 아이콘 세트(lucide 등)에 맞춰 재해석한다.

## 7. 접근성·상호작용

- 버튼, 링크, 입력 요소는 키보드 포커스가 보여야 한다. 포커스 색은 `--deep` 또는 `--blue` 계열을 사용한다.
- 아이콘 전용 버튼은 `aria-label` 또는 화면에 보이지 않는 라벨을 둔다.
- 표와 차트는 색만으로 상태를 구분하지 않는다. 라벨, 숫자, 패턴 중 하나를 함께 제공한다.
- 클릭 가능한 요소는 기본 40px 이상의 터치 영역을 확보한다.

## 8. 인쇄(PDF) 규칙

보고용 자료는 인쇄를 전제로 한다.

```css
@media print{
  @page{size:A4 landscape;margin:8mm}
  .topbar,.sidebar{display:none}
  .card{box-shadow:none;border:1px solid var(--line)}
  thead{display:table-header-group} tbody tr{break-inside:avoid}
  *{-webkit-print-color-adjust:exact;print-color-adjust:exact}
}
```

- 상단바에 `🖨 인쇄 / PDF` 버튼(`window.print()`)을 둔다.
- 제안서처럼 세로 문서가 기본인 경우 `A4 portrait`로 프로젝트에서 재정의한다.

## 9. 새 프로젝트 적용 순서

1. 이 폴더의 [fonts/](fonts/)를 프로젝트의 `web/fonts/`로 복사한다.
2. `web/styles/theme.css`에 표준 `:root` 토큰과 `@font-face`를 먼저 둔다.
3. `web/styles/app.css`에는 컴포넌트 스타일만 작성한다.
4. 하드코딩 색상이 필요해 보이면 먼저 새 토큰이 필요한지 검토한다.
5. 아이콘 버튼의 크기, radius, 정렬선, hover/active 상태를 같은 화면 안에서 통일한다.
6. 화면을 만든 뒤 미리보기에서 콘솔 에러와 주요 뷰포트 스크린샷을 확인한다.

## 10. 프로젝트별 적용 현황

| 프로젝트 | 테마 상태 | 규칙 |
|---|---|---|
| 검진팀 대시보드 | **표준 원본** (`dashboard/web/styles/theme.css`) | 이 문서의 기준. 토큰 변경 시 이 문서도 함께 갱신 |
| 건강검진 제안서 자동생성기 | 표준과 동일 토큰 (대시보드 테마의 원조, `proposal_generator_beta/web/styles/theme.css`) | 표준 토큰 유지. 제안서 특성상 인쇄는 세로(A4 portrait) 기준 |
| 종합검진센터 스케쥴관리 | **자체 토큰 사용** (`application_beta/styles/theme.css`: `--blue:#2563eb`, `--page:#eef3f9`, Malgun Gothic 우선 스택 등) | 카파시룰(외과수술식 변경)에 따라 이 프로젝트 내 작업은 **기존 자체 토큰을 따른다**. 표준 테마로의 통일은 사용자가 요청할 때만 진행 |

- **새 프로젝트**는 이 문서의 표준 토큰으로 시작한다. 표준 폰트는 이 폴더의 [fonts/](fonts/)에 있으니 프로젝트의 `web/fonts/`로 복사하고, `theme.css`는 `검진팀 대시보드/dashboard/web/styles/theme.css`를 복사한다.
- 기존 프로젝트에서 작업할 때는 그 프로젝트의 `theme.css`가 우선이다. 화면 간 스타일이 다르면 새로 만들지 말고 기존 토큰·컴포넌트 클래스를 재사용한다.
