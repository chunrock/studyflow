# AGENTS.md - 교육플로우 자료생성 에이전트 운영 규칙

이 저장소는 Claude Code/Codex 하네스 구성을 문서화하고, 팀 에이전트와 하위 프로젝트 템플릿을 재사용 가능한 자료로 관리한다. 실제 `~/.claude` 설정을 직접 변경하기 전에는 이 저장소의 내용을 검토하고, 필요한 대상만 복사한다.

## 기본 원칙

- 모든 작업은 계획을 먼저 세우고 승인 후 실행한다.
- 되돌리기 어려운 변경은 실행 전에 변경 대상과 백업 방식을 명시한다.
- 기존 파일을 덮어쓸 때는 `.bak` 또는 날짜가 포함된 백업을 만든다.
- API 키, 토큰, 인증서, 개인 경로 같은 비밀정보는 문서에 저장하지 않는다.
- 작업 이력은 `history/YYYY-MM-DD_{주제-슬러그}.md` 형식으로 남긴다.

## 팀 운영 기준

전역 기본 팀은 `agents/teams/` 아래에 둔다.

| 팀 | 역할 |
| --- | --- |
| `proposal-team` | 제안서, RFP 분석, 국책과제 보고서, 산출물 작성 |
| `dev-team` | 코드 개발, API, 인프라, 테스트, CI/CD |
| `bizdev-team` | 시장조사, 경쟁분석, 사업화 전략, 영업지원 |
| `design-team` | 인포그래픽, 다이어그램, UI/UX, 발표자료 |
| `ops-team` | 일정관리, 내부 커뮤니케이션, 프로젝트 운영 |
| `qa-team` | 문서 검증, 코드 리뷰, 컴플라이언스 체크 |
| `project-dev` | 특정 프로젝트의 FE/BE 통합 개발 |
| `project-frontend-dev` | 특정 프로젝트의 프론트엔드 전용 개발 |

## 하위 프로젝트 기준

하위 프로젝트 템플릿은 `subprojects/`에 둔다. 실제 프로젝트에 적용할 때는 다음 값을 먼저 채운다.

- 프로젝트명
- 실제 작업 경로
- 프론트엔드/백엔드/실행기 경로
- 고정 미리보기 포트
- 사용하는 팀과 담당 에이전트
- 검증 명령과 산출물 위치

## 검증 기준

- JSON 파일은 생성 후 파싱 가능해야 한다.
- Markdown 문서는 링크와 경로 예시가 저장소 구조와 맞아야 한다.
- 실제 Claude 설정으로 반영할 때는 `~/.claude/settings.json`, `~/.claude/CLAUDE.md`, `~/.claude/teams/*`를 별도로 백업한 뒤 복사한다.
